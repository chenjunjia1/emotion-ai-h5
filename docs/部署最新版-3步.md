# 部署今天早上最新代码（3 步）

本机已 `npm run build` 通过。压缩包：**`emotion-ai-h5-上传GitHub.zip`**

线上地址：**https://www.emovalue.top**（Vercel 项目 **emotion-ai-h5**）

## 第 1 步：上传到 GitHub（网页一次最多 100 个文件）

GitHub 会提示 **「Try uploading fewer than 100 at a time」**，不能一次拖整个 `src`（有 133 个文件）。

### 方法 A：分批上传（推荐，不用装 Git）

1. 双击运行 **`scripts\打包上传GitHub-分批.bat`**
2. 会打开文件夹 **`github-upload-batches`**，里面有 5 个 zip：
   - `batch-app.zip`（51 个文件）
   - `batch-lib.zip`（51 个文件）
   - `batch-components.zip`（27 个）
   - `batch-contexts.zip`、`batch-hooks.zip`（各很少）
3. 打开 https://github.com/chenjunjia1/emotion-ai-h5 → 进入 **`src`**
4. **每一批单独上传一次**（Add file → Upload files）：
   - 解压 `batch-app.zip` → 只拖 **`app` 文件夹** → Commit：`feat: V1 app`
   - 解压 `batch-lib.zip` → 只拖 **`lib` 文件夹** → Commit：`feat: V1 lib`
   - 解压 `batch-components.zip` → 拖 **`components`** → Commit：`feat: V1 components`
   - 同理上传 **`contexts`**、**`hooks`**
5. 最后上传根目录文件：在 `src` 里 Add file，从本机 `E:\emotion-ai-h5\src\` 拖入 **`middleware.ts`**（只有 1 个文件）

### 方法 B：只传 2 批（最快）

在 `src` 目录分 2 次上传即可（每批 < 100）：

| 第几次 | 从本机拖入的文件夹 |
|--------|-------------------|
| 1 | `E:\emotion-ai-h5\src\app` + `src\components` + `src\contexts` + `src\hooks` + `middleware.ts` |
| 2 | `E:\emotion-ai-h5\src\lib` |

每次 Commit 后等网页传完，再进行下一批。

## 若 Vercel 报错 `Can't resolve '@alicloud/...'`

GitHub 根目录的 **`package.json` 还是旧的**，没有阿里云短信依赖。

1. 解压 **`github-upload-batches\batch-root-config.zip`**
2. 打开 https://github.com/chenjunjia1/emotion-ai-h5（**仓库根目录**，不是 src）
3. **Add file → Upload files** → 拖入这 4 个文件（覆盖旧文件）：
   - `package.json`
   - `package-lock.json`
   - `next.config.ts`
   - `tsconfig.json`
4. Commit：`fix: add alicloud sms dependencies` → 等 Vercel 重新部署

---

## 若 Vercel 构建报错 `Can't resolve '@/lib/...'`

说明 **`src/lib` 还没上传到 GitHub**（只传了 app/components 不够）。

1. 解压 **`github-upload-batches\batch-lib.zip`**
2. 打开 https://github.com/chenjunjia1/emotion-ai-h5 → 进入 **`src`**
3. **Add file → Upload files** → 拖入 **`lib` 整个文件夹**
4. Commit：`feat: V1 lib` → 等 Vercel 自动重新部署

---

## 第 2 步：等 Vercel 自动部署

1. 打开 https://vercel.com → **emotion-ai-h5** → **Deployments**
2. 会出现一条新的，等 **Building → Ready**（约 2～5 分钟）
3. 若没有新部署：点最新一条 **⋯ → Redeploy**

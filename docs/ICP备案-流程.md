# ICP 备案流程（国内长期运营）

> 备案是 **域名 + 服务器** 在工信部登记，与代码功能无关。下证后把备案号填到环境变量，全站页脚自动显示。

---

## 为什么要做

- 国内域名长期运营、微信/支付审核、应用市场上架，通常要求 **已备案**
- 未备案站点可能被接入商拦截或限制

---

## 建议顺序（配合你现有资源）

1. **先有个体户执照**（见 `docs/深圳个体工商户-网上办理步骤.md`）
2. **域名** `emovalue.top` 已完成实名
3. **服务器**：Vercel 海外节点 **无法** 做国内 ICP；国内备案需 **大陆服务器**（阿里云/腾讯云 ECS + 备案服务号）或 **国内静态托管**（如阿里云 OSS + CDN 等方案）
4. 在服务器接入商控制台提交 **ICP 备案**（通常 7～20 个工作日）

---

## 备案通过后 — 代码只需 1 步

Vercel / `.env.local` 增加：

```env
NEXT_PUBLIC_ICP_BEIAN=粤ICP备2025xxxxxx号
```

Redeploy 后，所有带 `AppShell` 的页面底部会显示备案号（`src/components/layout/site-footer.tsx`）。

也可填工信部查询链接（以 `https://` 开头则渲染为可点击链接）。

---

## 备案材料清单（参考）

| 材料 | 说明 |
|------|------|
| 营业执照 | 个体户或企业 |
| 法人身份证 | 正反面 |
| 域名证书 | 在域名注册商下载 |
| 网站负责人 | 通常即法人，需核验手机号 |
| 网站名称 | 与 App 名称一致或相近，如「AI短视频运营灵感」 |
| 网站内容 | 选「其他」或「工具类」，描述为 AI 内容创作 H5 |

---

## 与当前 Vercel 部署的关系

| 部署方式 | 备案 |
|----------|------|
| 仅 Vercel（海外） | 不能做国内 ICP；可先用海外访问，国内体验与合规受限 |
| 国内 CDN 回源 Vercel | 部分厂商支持，以接入商政策为准 |
| 迁到国内服务器 | 标准备案流程，推荐长期方案 |

## 自检

```bash
npm run check:env
# ℹ️ NEXT_PUBLIC_ICP_BEIAN 未填 → 正常，下证后再填
# ✅ ICP 备案号已配置 → 已生效
```

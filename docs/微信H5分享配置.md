## 零、登录失败？「没有可登录账号」怎么办

### 你截图里的情况

提示 **「该微信还未注册公众平台账号」** 表示：

> 你用来扫码的这个微信号，**还没有被任何公众号绑定**（不是管理员，也不是运营者/关联员）。

这和「只有关联员权限、权限不够」略有不同——**第一步要先能登录进某个公众号的后台**。

### 公众号里常见角色（从低到高）

| 角色 | 能否登录 mp.weixin.qq.com | 能否配业务域名 / JS 域名 |
|------|---------------------------|---------------------------|
| **管理员** | ✅ | ✅ |
| **运营者**（常叫关联运营人员） | ✅ | ✅（一般够用） |
| **开发者** | ✅ | 偏接口开发，域名也在设置里 |
| **数据分析者** | ✅ | ❌ 通常不能改功能设置 |

配 **业务域名、JS 接口安全域名、下载校验文件**，用 **管理员** 或 **运营者** 登录即可，**不一定必须是注册时的那个管理员**。

---

### 情况 A：公司/朋友已经有一个公众号（最常见）

你需要 **管理员** 把你加为 **运营者**：

1. 让已有 **管理员** 用能登录的微信扫码进入 https://mp.weixin.qq.com  
2. **设置 → 人员设置 → 运营者管理**（或「公众号运营者」）  
3. **绑定运营者微信号** → 输入你的微信号（微信「我」→ 微信号那一栏）  
4. 你微信会收到 **公众平台安全助手** 的邀请，点 **同意**  
5. 再用你的微信 **重新扫码登录**，应能看到该公众号

若管理员找不到入口：在后台顶部 **搜索「运营者」**。

**你要发给管理员的一句话：**

> 请你在公众号后台「设置 → 人员设置 → 运营者管理」里，把我的微信号 xxx 绑成运营者，我这边要配置 H5 业务域名。

---

### 情况 B：没有任何人注册过公众号（你要自己注册）

1. 打开 https://mp.weixin.qq.com  
2. 点右上角 **立即注册** → 选 **订阅号** 或 **服务号**  
   - **订阅号**：个人/个体户可注册，审核快，配域名、分享一般够用  
   - **服务号**：通常要企业/个体户执照，能力更全，注册稍复杂  
3. 按流程填邮箱、主体信息（个体户执照等）、管理员微信扫码  
4. 审核通过后（订阅号常 1～3 天），用**管理员微信**登录  
5. 再按本文 **第三节** 配业务域名、下载校验文件

个体户执照办理见：`docs/深圳个体工商户-网上办理步骤.md`

---

### 情况 C：你说有「关联员」但仍登录不了

请核对：

| 检查项 | 说明 |
|--------|------|
| 是否扫对微信 | 必须用 **被绑定的那一个** 微信扫码（别用分身/小号） |
| 是否点了同意 | 绑定邀请要在 **公众平台安全助手** 里点同意 |
| 关联的是不是小程序 | 小程序后台是 mp.weixin.qq.com 里另一个入口，和 **公众号** 不是同一个 |
| 是否已过期 | 运营者绑定有时效，过期需管理员重新绑 |

仍不行 → 让管理员在 **人员设置** 里看你的微信号是否在列表中。

---

### 暂时不注册公众号，分享能不能用？

| 能力 | 没有公众号 |
|------|------------|
| 微信里转发链接，带标题+描述+图（读网页 og 标签） | **多数情况可以**，我们代码已配 `og-share.png` |
| 业务域名校验、JS-SDK 高级分享 | **不行**，必须登录公众平台 |
| 微信支付 | **不行**，需商户号 |

**结论**：若只想朋友转发链接时 **有标题有图**，可先部署现有代码试分享；**业务域名校验文件** 必须等能登录公众号后再做。

---


| 序号 | 做什么 | 在哪里做 | 要不要改代码 |
|------|--------|----------|--------------|
| A | 配置 **JS 接口安全域名** | 微信公众平台网页 | 不用 |
| B | 下载 **校验文件** 并放到网站 | 公众平台下载 + 你电脑 `public/` 文件夹 | **要放 1 个文件** |
| C | **部署上线** 让文件能被访问 | GitHub + Vercel | 推代码 / Redeploy |
| D | 在公众平台点 **确认** | 微信公众平台网页 | 不用 |
| E | 微信里试 **分享卡片** | 手机微信 | 不用 |

**代码侧我们已做好**：分享标题、描述、图片（`og-share.png`）都在 `layout.tsx` 里配好了。  
**你缺的主要就是 B + C + D**：把微信给的 `.txt` 文件放到网站根目录，让微信能访问到。

   - 你电脑上：`E:\emotion-ai-h5`  
   - 校验文件要放进：`E:\emotion-ai-h5\public\`

## 三、手把手：校验文件你要做什么（最重要）

### 第 1 步：进微信公众平台

1. 打开 https://mp.weixin.qq.com 并登录  
2. 左侧或顶部找到 **设置**  
3. 点 **公众号设置**  
4. 点 **功能设置**（有的界面叫「账号详情 → 功能设置」）

### 第 2 步：添加「业务域名」

1. 找到 **业务域名** 这一栏（不是「服务器域名」，那是小程序用的）  
2. 点 **设置** 或 **修改**  
3. 在输入框里填（**不要**写 `https://`，**不要**写后面的路径）：

   ```
   www.emovalue.top
   ```

4. 点 **下载校验文件**（或「申请验证」后下载）  
5. 浏览器会下载一个文件，名字类似：

   ```
   MP_verify_A1B2C3D4E5F6G7H8.txt
   ```

   > 每个人文件名里的字母数字 **不一样**，以你下载的为准。

6. **不要改文件名，不要改文件里面的内容**（通常文件里只有一行，内容和文件名里的那串字符一样）

### 第 3 步：把文件放进项目（你电脑上操作）

1. 打开文件夹：`E:\emotion-ai-h5\public\`  
2. 把下载的 `MP_verify_xxxx.txt` **复制粘贴** 到这个文件夹里  
3. 放好后，`public` 文件夹里应该同时有：

   ```
   E:\emotion-ai-h5\public\
   ├── og-share.png
   ├── favicon.svg
   └── MP_verify_A1B2C3D4E5F6G7H8.txt   ← 你刚放的（名字以你的为准）
   ```

4. 在项目根目录打开终端，检查是否识别到了：

   ```bash
   npm run check:env
   ```

   成功时会看到类似：

   ```
   ✅ 微信业务域名校验文件: public/MP_verify_A1B2C3D4E5F6G7H8.txt
   ```

   如果还是 ⚠️ 未找到，说明文件没放对位置或文件名不对。

### 第 4 步：把代码部署到 Vercel（让全网能访问这个文件）

微信是从 **公网** 访问你的文件，只放本地电脑不行，必须 **部署一次**。

**方式 A：GitHub 推送（推荐）**

1. 用 GitHub Desktop 或网页，把 `public/MP_verify_xxxx.txt` 一起提交上传  
2. 打开 Vercel → 项目 `emotion-ai-h5-v2`  
3. 等自动部署完成，或 **Deployments → Redeploy**

**方式 B：Vercel 网页直接上传（没有 Git 时）**

1. 在 GitHub 网页编辑，上传 `public` 里的 txt 文件  
2. Vercel 重新部署

### 第 5 步：浏览器验证文件能打开（必做）

部署完成后，用 **电脑浏览器** 新开一个标签页，访问（把文件名换成你的）：

```
https://www.emovalue.top/MP_verify_A1B2C3D4E5F6G7H8.txt
```

**正确结果**：页面只显示一行字母数字（和文件内容一样），**不是** 404，**不是** 跳转到首页。

**如果 404**：

| 可能原因 | 怎么办 |
|----------|--------|
| 还没部署 | Vercel 点 Redeploy，等 1～2 分钟再试 |
| 文件不在 `public/` | 必须放在 `public/` 根目录，不能放子文件夹 |
| 文件名被改了 | 改回微信下载时的原名 |
| 访问错了域名 | 业务域名填的是 `www.emovalue.top`，就用 www 访问 |

### 第 6 步：回微信公众平台点「确认」

1. 回到 **设置 → 公众号设置 → 功能设置 → 业务域名**  
2. 确认 `www.emovalue.top` 已添加  
3. 点 **确认** / **验证** / **保存**（按钮文案因版本可能略有不同）  
4. 显示 **验证成功** 或域名状态为 **已验证** 即可

## 四、JS 接口安全域名（顺手一起配）

仍在 **功能设置** 页面：

1. 找到 **JS接口安全域名**  
2. 点 **设置**  
3. 填写（同样不要 https）：

   ```
   www.emovalue.top
   ```

4. 按提示用 **管理员微信扫码** 确认  
5. （可选）若希望 `https://emovalue.top` 也能用，再单独加一条 `emovalue.top`

> JS 域名和「业务域名」是 **两个不同格子**，建议两个都配。

## 六、你不需要做的事（避免踩坑）

- ❌ 不要把校验文件内容改成别的  
- ❌ 不要把 `MP_verify_xxx.txt` 改成 `.html` 或别的后缀  
- ❌ 不要只放在桌面或 `docs/`，必须在 `public/`  
- ❌ 不要在微信里填 `https://www.emovalue.top/xxx` 带路径的域名  
- ❌ 校验文件 **不能** 只在本地 `localhost` 测通就算完成，必须线上 URL 能打开

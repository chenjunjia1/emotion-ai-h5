# 真·手机短信验证码（不是 1234）

## 你现在的情况

| 现象 | 原因 |
|------|------|
| 收不到短信 | Vercel 里 `SMS_PROVIDER=dev`，**不会真发短信** |
| 输入 1234 能登录 | 网站还在 **Mock 演示模式**（没开 `NEXT_PUBLIC_BACKEND_MODE=server`） |
| 有 60 秒倒计时 | 新代码已上线 ✅ |

你截图域名是 **`emotion-ai-h5.vercel.app`**（旧项目）。  
请优先用：**https://emotion-ai-h5-v2.vercel.app**（已配环境变量的那个）。

---

## 第一步：确认真登录模式（必做）

在 **emotion-ai-h5-v2** 的 Vercel 环境变量里必须有：

```
NEXT_PUBLIC_BACKEND_MODE=server
```

保存后 **Redeploy**。

**验证：** 登录页小字应是「验证码将发送到您的手机」，**不是**「Mock 验证码：1234」。  
且 **1234 不能再登录成功**（除非又去 Logs 里查了真验证码）。

开发阶段可保持：

```
SMS_PROVIDER=dev
```

此时验证码在 **Vercel → Deployments → Logs** 里，搜 `[sms:dev] code=xxxxxx`。

---

## 第二步：接入阿里云短信（收真短信）

### 1. 阿里云开通

1. 登录 https://www.aliyun.com
2. 开通 **短信服务**
3. 申请 **签名**（如你的 App 名）
4. 申请 **模板**（验证码类），模板变量名要是 **`code`**
5. 创建 **AccessKey**（RAM 子账号更安全）

### 2. Vercel 增加变量（emotion-ai-h5-v2）

| Key | Value |
|-----|--------|
| `SMS_PROVIDER` | `aliyun`（改掉 dev） |
| `ALIYUN_SMS_ACCESS_KEY_ID` | 你的 AccessKey ID |
| `ALIYUN_SMS_ACCESS_KEY_SECRET` | 你的 Secret |
| `ALIYUN_SMS_SIGN_NAME` | 短信签名名称 |
| `ALIYUN_SMS_TEMPLATE_CODE` | 模板 CODE，如 SMS_123456789 |

### 3. 上传最新代码并 Redeploy

需包含 `package.json` 里的阿里云 SDK 和更新后的 `src/lib/server/sms.ts`。

### 4. 测试

1. 打开 **emotion-ai-h5-v2** 网站
2. 填**真实手机号** → 获取验证码
3. 手机收到 6 位数字 → 填入登录

## 常见问题

**Q：换了 aliyun 还是没短信？**  
看 Vercel Logs 是否有 `[sms] aliyun send failed`，模板变量、签名审核是否通过。

**Q：还想内测，不想花钱？**  
保持 `SMS_PROVIDER=dev` + `BACKEND_MODE=server`，从 Logs 复制 6 位验证码，不要用 1234。

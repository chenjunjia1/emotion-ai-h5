# 入库验收清单（Supabase + 线上）

## 1. 先跑迁移（Supabase SQL Editor）

按顺序执行，详见 `docs/执行迁移005.md`、`docs/执行迁移006.md`：

1. `002_v1_schema.sql`
2. `003_support_feedbacks.sql`（可选）
3. `004_performance_indexes.sql`（可选）
4. **`005_v1_product_invite_growth.sql`**
5. **`006_fix_generations_and_invite.sql`**

## 2. Vercel 环境变量

- `NEXT_PUBLIC_BACKEND_MODE=server`
- `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- 改完后 **Redeploy**

## 3. 功能 → 表对照

| 操作 | 应写入的表/列 |
|------|----------------|
| 登录 | `users` |
| 选题盲盒 | `generations.type=topic_box`，`user_daily_usage.topic_box_count` |
| 标题扭蛋 | `generations.type=title_gacha`，`user_daily_usage.title_gacha_count` |
| 发布包 | `generations.type=publish_pack`，扣 `used_count` |
| 测号 | `generations.type=account_test` |
| 复盘 | `generations.type=review` |
| 神回复 | `generations.type=reply` |
| 爆款打分 | `generations.type=score`，`viral_score_count` |
| 签到/任务 | `users.growth_xp`、`growth_tasks_done`、`streak_days` |
| 邀请注册 | `invite_records`，邀请人 `invite_count` + `bonus_quota` |
| 开邀请盲盒 | `users.invite_blind_box_count` 减 1 |
| 内容库页 | 来自 `generations`（`/api/me?sync=1`） |

## 4. 快速自测

1. 登录后抽一次选题盲盒 → Table Editor `generations` 有新行 `topic_box`
2. 打开「内容库」→ 与 `generations` 条数一致（约 1.2s 后自动同步）
3. 用邀请链接注册小号 → `invite_records` 有记录
4. 邀请页开盲盒 → `invite_blind_box_count` 减 1

若 `generations` 仍无新行：多半 **006 未执行**，`type` 被旧 CHECK 拦住（服务端日志会有 `[recordGeneration]`）。

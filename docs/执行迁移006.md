# Supabase 迁移 006（入库修复，推广前必做）

在 **002、005 已执行** 的前提下，在 SQL Editor 执行：

`supabase/migrations/006_fix_generations_and_invite.sql`

**逐步图文说明（005→006→Vercel→自测）**：见 [Supabase迁移005-006-详细步骤.md](./Supabase迁移005-006-详细步骤.md)

## 本迁移会修复

- `users.invite_count` — 邀请人数统计可写入
- `users.growth_tasks_done` — 今日任务完成记录入库
- `generations.type` — 支持 `publish_pack`、`topic_box`、`title_gacha`、`account_test`、`review` 等

## 执行顺序（生产）

1. `001_initial.sql`（可选）
2. **`002_v1_schema.sql`**
3. `003_support_feedbacks.sql`
4. `004_performance_indexes.sql`
5. **`005_v1_product_invite_growth.sql`**
6. **`006_fix_generations_and_invite.sql`** ← 本次

## 如何确认成功

- Table Editor → `users` 有列 `invite_count`、`growth_tasks_done`
- 线上抽选题 / 发布包后，`generations` 表有新行，`type` 为 `topic_box` / `publish_pack` 等
- 邀请注册后 `invite_records` 有记录，邀请人 `bonus_quota` 增加

执行后 **Vercel Redeploy**。

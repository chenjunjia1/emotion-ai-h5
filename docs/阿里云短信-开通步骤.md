# 阿里云真短信 — 开通步骤（照着做）

代码已接好，你只需在阿里云申请 + Vercel 填 5 个变量。

---

## 一、阿里云控制台（约 30 分钟～1 天，等审核）

### 1. 登录

https://www.aliyun.com → 登录（需实名）

### 2. 开通短信服务

搜索 **「短信服务」** → 开通

### 3. 申请签名

路径：**短信服务 → 国内消息 → 签名管理 → 添加签名**

| 填什么 | 示例 |
|--------|------|
| 签名名称 | `AI短视频运营助手` 或你的品牌名 |
| 适用场景 | 验证码 |
| 类型 | 按你主体选（企业/个人/小程序等） |

提交后等 **审核通过**（几小时～1 工作日）

### 4. 申请模板

路径：**模板管理 → 添加模板**

| 填什么 | 示例 |
|--------|------|
| 模板类型 | 验证码 |
| 模板内容 | `您的验证码为：${code}，5分钟内有效，请勿泄露。` |
| 变量 | 必须有 **`code`**（和代码里一致） |

记下审核通过后的 **模板 CODE**，形如：`SMS_493440350`

### 5. 创建 AccessKey

路径：**右上角头像 → AccessKey 管理**（建议用 RAM 子账号，只开短信权限）

记下：

- **AccessKey ID**
- **AccessKey Secret**（只显示一次，务必保存）

---

## 二、Vercel 环境变量（emotion-ai-h5-v2）

**Settings → Environment Variables**，新增或修改：

| Key | Value |
|-----|--------|
| `SMS_PROVIDER` | `aliyun` |
| `ALIYUN_SMS_ACCESS_KEY_ID` | 你的 ID |
| `ALIYUN_SMS_ACCESS_KEY_SECRET` | 你的 Secret |
| `ALIYUN_SMS_SIGN_NAME` | 签名名称（和审核通过的一致） |
| `ALIYUN_SMS_TEMPLATE_CODE` | 如 `SMS_493440350` |

**保留已有的：**

- `NEXT_PUBLIC_BACKEND_MODE` = `server`
- Supabase 三个 + `SESSION_SECRET`

三个环境 **Production / Preview / Development** 都勾选 → **Save**

---

## 三、上传代码并部署

确保 GitHub 上有最新：

- `package.json`（含 `@alicloud/dysmsapi20170525`）
- `src/lib/server/sms.ts`
- `src/components/modals/login-modal.tsx`
- `src/lib/i18n/zh.ts`

Commit 后等 Vercel **Ready**，或手动 **Redeploy**。

## 四、测试

1. 打开 **https://emotion-ai-h5-v2.vercel.app**（无痕窗口）
2. 登录页应显示：**验证码将发送到您的手机**（不是 Mock 1234）
3. 填 **真实手机号** → 获取验证码 → 等短信
4. 输入 **6 位数字** 登录

失败时：Vercel **Logs** 搜 `[sms] aliyun`，看错误码。

---

## 五、常见错误

| 日志/现象 | 处理 |
|-----------|------|
| `isv.SMS_SIGNATURE_ILLEGAL` | 签名名和 Vercel 里不一致或未过审 |
| `isv.SMS_TEMPLATE_ILLEGAL` | 模板 CODE 错，或变量不是 `code` |
| `isv.BUSINESS_LIMIT_CONTROL` | 同一号发太频繁，等 60 秒 |
| 仍显示 Mock 1234 | 用 v2 域名；确认 `BACKEND_MODE=server` 并 Redeploy |

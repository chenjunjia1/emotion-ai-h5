# 回归测试报告（2026-05-25）

## 变更范围

| 项 | 说明 |
|----|------|
| 发布包 | 3 步 → 2 步；平台改为单行 Chip；合并表单；去掉生成中占位大卡 |
| 热点卡片 | `ring-inset` + 首条 `border`，消除角上粉色残影 |
| 生成体验 | API 2.8s 客户端超时；服务端 2s 预算 mock 兜底；成功提示素材库位置 |
| AI 助手 | 选场景仅填入输入框，需点发送才生成 |
| 首页轮播 | 第 2 屏改为「盲盒 + 情绪助手」双入口 |
| 复盘联动 | 推荐选题 + 复盘建议写入发布包 URL 与补充说明 |

## 正向用例

| ID | 用例 | 预期 |
|----|------|------|
| P1 | `WIZARD_STEPS` 为 2 项 | 「填写选题与方向」「生成内容」 |
| P2 | 发布包单页含平台 Chip + 方向 + 生成按钮 | 无「下一步：选择内容方向」 |
| P3 | 复盘页点击「用推荐选题生成发布包」 | URL 含 `from_review=1`、`review_hint` |
| P4 | 从复盘进入发布包 | 顶部提示「已带入复盘…」且 `extraNote` 含建议 |
| P5 | 生成成功 | Toast 提示下方与「生成记录」；成功条显示素材库 |
| P6 | 首页轮播第 2 屏 | 盲盒 → `/topic-box`；助手 → `/emotion-chat` |
| P7 | AI 助手点场景卡 | 仅填输入框 + Toast，不出现「AI 正在想」 |
| P8 | AI 助手点发送 | 才开始生成并展示结果 |
| P9 | 热点 TOP 卡片四角 | 无多余粉色弧形外框 |

## 反向用例

| ID | 用例 | 预期 |
|----|------|------|
| N1 | 发布包不应再出现三步「选择平台」步骤条 | 失败若文案仍为旧版 |
| N2 | 未点发送仅选场景 | 不应消耗灵感、不应出结果 |
| N3 | 生成超时（>2.8s 无响应） | Toast「生成超时」且保留表单 |
| N4 | 复盘链接仅 `topic` 无 `review_hint` | 仍可生成，但无联动提示条 |
| N5 | 首页轮播不应再嵌「1分钟发布包」整屏 | 失败若仍为 `publishPack` 屏 |

## 自动化执行

```bash
npm run typecheck
node scripts/test-regression-pack.mjs
```

## 手工建议（上线前）

1. 登录后走一遍：热点 → 改成我的文案 → 生成（应在约 2s 内出结果）。
2. 复盘 → 生成报告 → 用推荐选题生成发布包 → 确认选题与建议一致。
3. 首页左右滑轮播，确认第 2 屏为盲盒/助手双卡。
4. AI 助手：点「播放太低咋办」→ 确认未生成 → 点发送 → 再出结果。
